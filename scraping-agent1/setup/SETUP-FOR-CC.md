# scraping-agent Setup For Claude Code

このファイルは、`scraping-agent` の初期設定手順書です。

Claude Code はこのファイルを見ながら次の順で進行する。
ユーザーは Claude Code の案内に従えばよい(手動でファイル編集はしない)。

---

## 前提

- 認証設定は `.env` に置く
- Google OAuth のトークン保存先は `credentials/tokens.json`
- スプレッドシートIDは `.env` の `SPREADSHEET_ID` に保存する
- API キーはすべて `.env` に保存する(ユーザーには触らせず、エージェントが書き込む)

### 必須/任意の区分

| 項目 | 区分 | 用途 / 影響 |
|---|---|---|
| 依存パッケージ(`node_modules/`) | **必須** | 何も動かない |
| `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` | **必須** | Sheets / Drive への書き込みに必須 |
| `credentials/tokens.json` | **必須** | OAuth 認可の結果 |
| `SPREADSHEET_ID` | **必須** | 書き込み先。`create-template-sheet.mjs` で自動設定可 |
| `BRAVE_SEARCH_API_KEY` | **任意(推奨)** | `search-based` モード(検索経由のリサーチ)で必要。無くても `list-detail` / `single` / `places` は使える |
| `GOOGLE_PLACES_API_KEY` | **任意(推奨)** | `places` モード(店舗の電話番号取得)と `enrich-contact --use-places` で必要 |

> **方針**: 必須だけで動くが、**おすすめは最初に両方の任意キーも揃えてしまうこと**(後から都度追加するのは面倒)。ユーザーが迷っていたら「両方設定することを推奨します」と案内する。

---

## Claude Code が最初に確認すること

1. `package.json` が存在すること
2. `node_modules` が存在すること(Puppeteer 同梱の Chrome も含む)
3. `.env` が存在し、**必須キー** が入っていること:
   - `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET`(空でないこと)
   - `SPREADSHEET_ID`(空でないこと)
4. `credentials/tokens.json` が存在すること
5. 以下の **任意キー** の充足状況を別途把握する(欠けていても先に進めるが、後で案内する):
   - `BRAVE_SEARCH_API_KEY`
   - `GOOGLE_PLACES_API_KEY`

必須が全て揃っていればセットアップは「最低限完了」とみなしてよい。
任意が欠けている場合は、ユーザーに「今すぐ追加しますか? それとも後で?」と選ばせる(後述)。

## 進行フロー

初回ユーザー向けに、**冒頭で全体像を見せてから** 始める:

```text
これから初期設定を進めます。

必須ステップ(5〜10分):
  1. 依存パッケージのインストール
  2. Google OAuth クライアントの発行
  3. Google 認証(ブラウザで「許可」)
  4. スプレッドシートの自動作成

任意ステップ(推奨、+3〜5分):
  5. Brave Search API キーの取得(検索経由のリサーチ用)
  6. Google Places API キーの取得(店舗の電話番号取得用)

最後に、定期実行の識別子(launchd ラベル)を確認します(多くの場合デフォルトのままでOK)。

任意の2つは、使う予定がなければ今スキップしても、後から追加できます。

どう進めますか?
1. 全部まとめてセットアップする(推奨)
2. 必須だけ先にセットアップする(任意は後でやる)
3. すでに一部終わっている — 状態を見て不足分だけやる
```

- `1` が選ばれた場合 → すべてのステップを順に実行(任意も実行)
- `2` が選ばれた場合 → 必須ステップだけ実行し、Step 6 で任意キー追加のお誘いを出して完了
- `3` が選ばれた場合 → 状態チェックに従って未完了だけ進める(既に揃っているものはスキップ)

### 1. 依存関係のインストール(必須)

`node_modules` がなければ `npm install` を実行する。

**注意**: Puppeteer が Chrome バイナリをダウンロードするため、回線速度によっては数分かかる。
ユーザーにあらかじめ伝えておく:

```text
これから依存パッケージをインストールします。
Puppeteer という自動ブラウザツール(300MB程度)を含むため、
回線によっては2〜5分かかることがあります。そのままお待ちください。
```

完了したら次へ。

### 2. `.env` の確認

`.env` がなければ、まず `.env.example` をコピーして作成:

```bash
cp .env.example .env
```

その後、必須の **Google OAuth → 任意の Brave / Places** の順で値を埋めていく。

### 2-A. Google OAuth クライアントを発行する(必須)

Google Sheets API と Drive API を使うため、OAuth クライアントが必要。
ステップ A-1 〜 A-5 を **1つずつ** 案内する。一度に全部見せない。

#### A-1. GCPプロジェクトを作成する

```text
Google Cloud Console でプロジェクトを作ります。

1. https://console.cloud.google.com/ を開く
2. 画面上部のプロジェクト選択 → 「新しいプロジェクト」をクリック
3. プロジェクト名は「scraping-agent」などでOK
4. 「作成」をクリック
5. 作成後、画面上部で今作ったプロジェクトが選択されていることを確認

できたら「できた」と教えてください。
```

#### A-2. APIを有効にする

```text
次に、必要なAPIを有効にします。2つあるので順番にやっていきます。

1. 画面上部の検索バーに「Google Sheets API」と入力 → 「有効にする」をクリック
2. 同じように「Google Drive API」を検索 → 「有効にする」

2つ全部できたら「できた」と教えてください。
```

#### A-3. OAuth同意画面を設定する

```text
次に、OAuthの設定をします。

1. 画面上部の検索バーに「Google Auth Platform」と入力してクリック
   (または https://console.cloud.google.com/auth/overview を直接開く)
2. 「開始」または「始める」ボタンが表示されたらクリック
3. 以下を入力:
   - アプリ名: 「scraping-agent」
   - ユーザーサポートメール: 自分のGmailアドレスを選択
4. 対象(ユーザータイプ)は「外部」を選択
5. 連絡先のメールアドレス: 自分のGmailアドレスを入力
6. 同意のチェックボックスにチェックを入れて「作成」をクリック

できたら「できた」と教えてください。
```

#### A-4. テストユーザーを登録する

```text
次に、自分のGoogleアカウントをテストユーザーとして登録します。
これをしないと、この後の認証でブロックされてしまいます。

1. Google Auth Platform の左メニューから「対象」をクリック
   (または https://console.cloud.google.com/auth/audience を直接開く)
2. 「テストユーザー」セクションの「+ Add users」をクリック
3. 自分のGmailアドレスを入力
4. 「保存」をクリック

できたら「できた」と教えてください。
```

#### A-5. OAuthクライアントを作成する

```text
次に、クライアントIDとシークレットを発行します。

1. Google Auth Platform の概要ページで「OAuthクライアントを作成」をクリック
   (または左メニューの「クライアント」→ 上部の「+ OAuthクライアントを作成」)
2. アプリケーションの種類: 「デスクトップ アプリ」を選択
3. 名前: 「scraping-agent-local」などでOK
4. 「作成」をクリック
5. ダイアログに「クライアント ID」が表示されるのでコピー
6. 次に、クライアント一覧から今作ったクライアント名をクリック
7. 詳細画面の右下にある「クライアント シークレット」をコピー

⚠️ クライアントシークレットは後から再表示できません。
   必ずこのタイミングでコピーしてください。

クライアントIDとクライアントシークレットの両方を、
このチャットにそのまま貼り付けてください。
こちらで自動的に `.env` に書き込みます。
```

**エージェントの処理**:
ユーザーが貼り付けた値を受け取ったら、`.env` の該当行を更新する。ユーザーに直接編集させない。

```env
GOOGLE_CLIENT_ID=ユーザーが貼り付けた値
GOOGLE_CLIENT_SECRET=ユーザーが貼り付けた値
```

### 2-B. Brave Search API キーを取得する(任意・推奨)

「検索エンジン経由のリサーチ」機能(例: `「新宿 居酒屋 ホームページ」で検索→ヒットを巡回`)に使う。
無料プランで月2000クエリまで利用可能。

**事前の案内**:

```text
次に Brave Search API を設定します。
これは search-based モード(「○○ ホームページ」で検索してヒット先を巡回)で使います。
無料プランで月 2,000 クエリまで無料です。

使わなくても list-detail / single / places モードは動きますが、
ヒアリング時に「ホームページから集めたい」系のパターンが多いので、
設定しておくことを強くおすすめします。

設定しますか?
1. する(推奨)
2. スキップする(後で追加する)
```

「スキップ」を選ばれたら `.env` の `BRAVE_SEARCH_API_KEY` は空のままでよい。

「する」を選ばれた場合、以下を案内する:

```text
Brave Search API キーを取得します。

1. https://brave.com/search/api/ を開く
2. 「Get started free」または「API Keys」ボタンをクリック
3. メールアドレスでサインアップ(または Google ログイン)
4. ダッシュボードで「Subscriptions」→ 無料プラン「Free」を選択して登録
5. ダッシュボードの「API Keys」でキーを作成 → コピー

キーを取得したら、このチャットにそのまま貼り付けてください。
```

**エージェントの処理**: `.env` の `BRAVE_SEARCH_API_KEY` に書き込む。

> **⚠️ クレジットカード登録について**:
> Free プランの場合、クレジットカード登録が必要になることがあるが、無料枠を超えた時点で自動停止されるので料金は発生しない。不安なユーザーには「Subscriptions」画面で使用量上限を確認してもらう。

### 2-C. Google Places API キーを取得する(任意・推奨)

「飲食店・美容室・医療機関・小売店などの電話番号を取得」したい場合、
Web スクレイピングより Places API の方が **圧倒的に簡単かつ合法** なので、こちらを使う。

**対象ユーザー**: 営業先リスト(店舗系)/ BtoB でも Google Maps に登録がある業種 を扱う人

**事前の案内**:

```text
次に Google Places API を設定します。
これは places モード(「○○区の居酒屋の電話番号一覧」のように店舗情報を一気に取得)と、
連絡先エンリッチの Places フォールバックで使います。

ステップ 2-A で作った GCP プロジェクトにもう1つ API を追加する形です。
料金は Maps Platform の月 $200 無料クレジット内で十分おさまります(数千件まで無料目安)。

使わなくても他のモードは動きますが、店舗の電話番号を合法的に取れるのは
このモードだけなので、設定しておくのがおすすめです。

設定しますか?
1. する(推奨)
2. スキップする(後で追加する)
```

「スキップ」を選ばれたら `.env` の `GOOGLE_PLACES_API_KEY` は空のままでよい。

「する」を選ばれた場合、以下を **1つずつ** 案内する:

#### C-1. Places API (New) を有効化する

```text
ステップ 2-A で作った GCP プロジェクトに戻ります。

1. https://console.cloud.google.com/ を開く
2. 画面上部で「scraping-agent」プロジェクトが選択されていることを確認
3. 画面上部の検索バーに「Places API (New)」と入力 → クリック
   (検索候補に「Places API」と「Places API (New)」の両方が出たら、必ず【(New)】の方を選ぶ)
4. 「有効にする」をクリック
5. 有効化が完了したら「できた」と教えてください。

※ 「(New)」ではない旧 Places API も存在しますが、このシステムは (New) を使います。
```

#### C-2. API キーを発行する

```text
次に API キーを発行します。

1. 左メニュー → 「APIとサービス」 → 「認証情報」を開く
   (または https://console.cloud.google.com/apis/credentials を直接開く)
2. 上部の「+ 認証情報を作成」 → 「API キー」をクリック
3. API キーが発行されるのでコピー
4. (推奨)「キーを制限」をクリックし、「API の制限」で
   「Places API (New)」のみにチェックを入れて保存
   → これで万が一キーが漏れても、他のAPIでは使えないので料金事故を防げる

コピーした API キーを、このチャットにそのまま貼り付けてください。
```

**エージェントの処理**: `.env` の `GOOGLE_PLACES_API_KEY` に書き込む。

#### C-3. Cloud Billing を有効化する(未設定の場合)

Places API は有料 API のため、プロジェクトに請求先アカウントの紐付けが必要。
(月 $200 の無料クレジットを使うにも、Billing は有効化されていないといけない)

> 既に他のプロジェクトで Billing を設定済みの場合は、このプロジェクトに同じ請求先アカウントを紐付けるだけでよい。

```text
API キーを使うには、Cloud Billing(請求先アカウント)をプロジェクトに紐付ける必要があります。

1. 左メニュー → 「お支払い」を開く
   (または https://console.cloud.google.com/billing/linkedaccount を開く)
2. 「請求先アカウントをリンク」をクリック
3. 既存の請求先アカウントがあればそれを選ぶ
   なければ「請求先アカウントを作成」からクレジットカードを登録

※ 月 $200 の Maps Platform 無料クレジットは自動で適用されます。
  通常のリサーチ用途(数千件/月)なら料金は発生しません。

できたら「できた」と教えてください。
```

#### C-4. 動作確認

API キー設定直後に、簡単なテストを実行する:

```bash
# 東京駅周辺のカフェを3件だけ取得してみる(ドライラン)
echo '{"mode":"places","query":"東京駅 カフェ","maxItems":3}' | node tools/research.mjs --config - --dry-run
```

3件分の店名・住所・電話番号が表示されれば成功。
`401 / 403` エラーが出た場合は:
- Places API (New) が有効化されているか
- Billing が紐付いているか
- API キーの制限が正しく「Places API (New)」を含んでいるか

を順に確認する。

### 4. スプレッドシートの準備(必須)

この agent はリサーチごとに動的にシートを作るので、事前に中身を用意しておく必要はない。
空のスプレッドシートを Claude Code が自動で作成する:

```bash
node tools/create-template-sheet.mjs
```

実行すると:
1. 新しいスプレッドシートが Drive に作られる(タイトル: `scraping-agent リサーチ`)
2. README シートが1枚用意される(使い方の簡易説明つき)
3. 作成されたスプレッドシートの URL と ID、および次に実行すべきコマンドが表示される

**エージェントの処理**: スクリプトが表示した `spreadsheetId` を受け取り、続けて以下を実行して `.env` に保存する:

```bash
node tools/init-spreadsheet.mjs --id <表示されたID>
```

これで接続テストまで自動で通る(成功すれば ✓ 表記が出て、シート数が報告される)。

ユーザーへの案内例:

```text
今から新しいスプレッドシートを作成して、.env に接続情報を保存します。
Googleドライブに「scraping-agent リサーチ」という名前で追加されます。
(少し待ってください...)
```

> **補足**: 既に使いたいスプレッドシートがある場合は以下のコマンドだけでOK:
>
> ```bash
> node tools/init-spreadsheet.mjs --id <SPREADSHEET_ID または URL>
> ```

### 5.5. 定期実行の識別子を確認(任意)

このツールは macOS の launchd 経由で「毎朝9時に自動リサーチ」などの定期実行ができる。
その際、各スケジュールは以下の形式の識別子(ラベル)で登録される:

```
<LAUNCHD_LABEL_PREFIX>.<スクリプト名>
→ 例: local.scraping-agent.新宿_居酒屋.plist が ~/Library/LaunchAgents/ に作られる
```

`.env` の `LAUNCHD_LABEL_PREFIX` を変更すれば、このプレフィックスを自分用に変えられる。
通常はデフォルト(`local.scraping-agent`)のままで問題ない。

ユーザーへの案内:

```text
このツールは macOS の機能(launchd)で定期実行ができます。
その際、各スケジュールに以下のラベルが付きます:

  デフォルト: local.scraping-agent.<スクリプト名>
  (例: ~/Library/LaunchAgents/local.scraping-agent.新宿_居酒屋.plist)

通常はデフォルトのままで問題ありません。
他のツールと衝突しそうな場合や、会社名・個人名を入れたい場合は変更できます。

変更しますか?

1. デフォルト(local.scraping-agent)のままでOK
2. 変更する(希望のプレフィックスを教えてください。例: com.myteam.research)
```

「2」を選ばれた場合、ユーザーから受け取った値を `.env` の `LAUNCHD_LABEL_PREFIX` に書き込む。
(受講生にファイルを直接触らせず、エージェント側で更新する)

> 逆ドメイン形式(`com.example.<name>` や `local.<name>`)を推奨。
> 数字・英字・`.` `-` `_` のみ使用可。空白や特殊文字は避ける。

### 6. 完了

完了時の伝え方は、**任意キーの充足状況で 2 パターン** に分ける。

#### 6-A. 必須+任意すべて揃っている場合

```text
セットアップが完了しました!

使える機能: places / search-based / list-detail / single の 4 モード、連絡先エンリッチ、定期実行

使い方:
- 「リサーチして」と言えば、対話しながらスクレイピングを実行できます
- 「営業先リスト作って」「競合調査したい」などの発話でもOK

最初のおすすめ:
- 試しに「テストで https://example.com から見出しを取ってみて」
  と言ってみると、軽い動作確認になります

困ったときは「CLAUDE.md を見せて」と言ってください。
```

#### 6-B. 必須のみ揃って、任意が欠けている場合

```text
必須のセットアップが完了しました!

今すぐ使える機能: list-detail / single モードのリサーチ、スケジュール、スクリプト化
今は使えない機能:
```

## よくあるエラーと対応

| エラーメッセージ | 原因 | 対応 |
|---|---|---|
| `GOOGLE_CLIENT_ID / GOOGLE_CLIENT_SECRET が未設定です` | `.env` がないか値が空 | ステップ 2-A の手順を案内 |
| `Token file not found` | Google 認証未完了 | ステップ 3 の認証フローを案内 |
| `SPREADSHEET_ID not set` | スプシ未設定 | ステップ 4 を案内 |
| `BRAVE_SEARCH_API_KEY is not set` | Brave キー未設定 | ステップ 2-B を案内(任意) |
| `GOOGLE_PLACES_API_KEY が未設定です` | Places キー未設定 | ステップ 2-C を案内(任意) |
| `アクセスがブロックされました` | テストユーザー未登録 | ステップ A-4 を案内 |
| `Request had insufficient authentication scopes` | スコープ不足 | `node auth-google.mjs` を再実行 |
| `robots.txt によって ... アクセス禁止` | サイト側の規約違反 | 別のサイトでリサーチするよう案内 |
| `Brave Search API error: 401` | API キーが間違い | ステップ 2-B でキーを再発行 |
| `Brave Search API error: 429` | 月次クオータ超過 or 秒間QPS超 | 翌月まで待つ、または有料プランを検討 |
| `Google Places API 認証/権限エラー(401/403)` | Places API (New) 未有効化 or Billing 未設定 or API キーの制限で弾かれている | ステップ C-1 / C-3 / C-2 の順に確認 |
| `Google Places API エラー 429` | レート制限 or クォータ超過 | クォータ確認、または時間を置いて再試行 |

## 運用メモ

- 初回は必ずこのファイルの順で進める
- ユーザーには毎回「次に何をすればよいか」を1ステップずつ伝える
- ユーザーにファイルを直接編集させない。チャットに値を貼り付けてもらい、エージェントが `.env` を生成・更新する
- セットアップ途中で停止しても、途中から再開できる設計(状態チェックで不足分だけ案内する)
- **任意キー(Brave / Places)は後から追加できる**。ユーザーが「今はいらない」と言ったら、スキップして `完了(6-B パターン)` で終える
