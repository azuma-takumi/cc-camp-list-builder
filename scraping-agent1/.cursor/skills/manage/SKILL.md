# 管理スキル

保存済み資産(スクリプト/スケジュール/リサーチシート/ログ)を見る・消す・整理するためのスキル。
破壊的な操作(削除)は必ず最終確認を挟む。

---

## トリガーとワークフロー分岐

ユーザーの発話で分岐する:

### ケース A: 一覧を見せる系

発話例: 「保存してあるスクリプト見せて」「スケジュール一覧」「過去のリサーチ」

```text
何を見ますか?

1. 保存済みスクリプト(scrapers/ の中身)
2. 登録済みの定期実行スケジュール
3. 過去のリサーチシート(スプシ内)
4. 実行ログ(定期実行の最新ログ)
```

各選択肢で以下を実行:

#### 1. 保存済みスクリプト一覧

```bash
node tools/run-scraper.mjs --list
```

#### 2. 登録済みスケジュール一覧

```bash
node tools/list-schedules.mjs
```

#### 3. 過去のリサーチシート一覧

```bash
node tools/research-helpers.mjs list-sheets
```

#### 4. 実行ログを見る

ログは `~/Library/Logs/scraping-agent/<名前>.log` にある。

```bash
# どのスクリプトのログを見たいか番号で聞いてから
ls -t ~/Library/Logs/scraping-agent/*.log 2>/dev/null
# 最新100行を表示
tail -n 100 ~/Library/Logs/scraping-agent/<名前>.log
# 失敗ログ
tail -n 100 ~/Library/Logs/scraping-agent/<名前>.err.log
```

### ケース B: 削除系

発話例: 「スケジュール削除して」「このスクリプト消して」「<名前>やめたい」

#### B-1. スケジュール削除

```text
どのスケジュールを削除しますか?

<list-schedules の結果を番号付きで列挙>
N. キャンセル
```

ユーザーが番号で選んだら最終確認:

```text
以下を削除します。よろしいですか?

- スケジュール: <名前>
- 削除対象: ~/Library/LaunchAgents/<LAUNCHD_LABEL_PREFIX>.<名前>.plist(デフォルトは local.scraping-agent)
- スクリプト本体は残ります(scrapers/<名前>.mjs)

1. 削除する
2. キャンセル
```

実行:

```bash
node tools/schedule.mjs --name "<名前>" --remove
```

#### B-2. スクリプト削除

```text
どのスクリプトを削除しますか?

<scrapers 一覧を番号付きで>
N. キャンセル
```

最終確認:

```text
以下を削除します。よろしいですか?

- スクリプト: scrapers/<名前>.mjs
- 関連する定期実行スケジュール: <あれば表示、なければ「なし」>

1. スクリプトだけ削除
2. スクリプト + スケジュール両方削除
3. キャンセル
```

実行:

```bash
# スケジュールも消す場合は先に
node tools/schedule.mjs --name "<名前>" --remove
# 本体削除
rm scrapers/<名前>.mjs
```

**注意**: スプシ側のシートは消さない。シート自体を消したいならユーザーにスプシを直接開いてもらう。

### ケース C: 実行ログを確認する系

発話例: 「昨日の定期実行ちゃんと走った?」「エラー出てる?」

1. スケジュール一覧を取得

```bash
node tools/list-schedules.mjs --json
```

2. ユーザーにどのスクリプトの状況を見たいか聞く

3. 最新のログを確認

```bash
tail -n 50 ~/Library/Logs/scraping-agent/<名前>.log
tail -n 50 ~/Library/Logs/scraping-agent/<名前>.err.log
```

4. 結果をサマリー報告

```text
<名前> の実行状況:

- 最終実行: <ログの最終タイムスタンプ>
- 直近の結果: 成功 <N>件 / 失敗 <N>件
- エラーログ: <あれば内容を要約、なければ「なし」>

次にできること:
1. 今すぐ1回実行して動作確認する
2. 完全なログを見る
3. スケジュールを変更する
4. 終了
```

---

## 破壊的操作の絶対ルール

- **削除系は必ずメニュー形式の最終確認** を挟む(ユーザーが番号を選ぶまで実行しない)
- **削除範囲を明示**(スケジュールだけ / スクリプトも含む / シートは含まない)
- **複数を一括削除** したいときは、一つずつ番号で選ぶ or 明示的に「全て」と確認する
- **シートの削除は行わない**(データが消える危険があるため、ユーザーに直接スプシを開いてもらう)
