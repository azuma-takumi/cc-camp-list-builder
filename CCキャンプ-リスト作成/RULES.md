# 作業ルール — スポーツ用品業界：メールアドレス

この案件のスクリプト操作・シート編集で守るべきルール一覧。
過去の事故・修正ログから抽出。破ると手戻りが大きいものを優先して記載。

---

## シート操作

### A列は読み取り専用（編集は一切禁止）
- A列（No）は色付きセルで問題行を検知する仕様になっている
- **スクリプトからの書き込みは絶対禁止**。`updateRows(SHEET, row, 0, ...)` を呼ぶのは禁止
- `batchUpdate` による書式変更も禁止
- 書き込みは必ず **B列（index=1）から開始**する
- A列の連番確認は `readSheetValues` で読み取るだけでよい。修正が必要な場合はユーザーが手動で行う

### 東たくみ以外の行は一切触らない
- 他の記入者（emiyau、ゆうさつき 等）の行は読み取り専用
- 全スクリプトで `writer !== "東たくみ" → continue` を必ず入れる
- `repair-youtube-sales-rows` のように行範囲を手動指定するスクリプトも例外なく適用

### コピーシートに書き込み・オリジナルシートは重複チェックのみ
| 用途 | スプレッドシートID |
|---|---|
| 書き込み先（コピー） | `1E7sL6TjDiGWUF77uMAc88XK7OzXXS8wgDgwInI5Ad1c` |
| 重複チェック（オリジナル） | `1WG00opfjyNsUO6Apr-IEbH1KxmDlYyaGjPoV6LDnJd0` |

- コピーシートだけ見ると他の作業者が追加した行が漏れる → **重複チェックは必ずオリジナルシートで行う**
- オリジナルへの転記は、コピーシートで確認が取れた行だけをまとめて追記する
- 追記位置は「オリジナルシートのB列に値がある最終行の次」

---

## 重複チェック

- **チャンネル名のみ**で重複判定する
- メールアドレスが同じでもチャンネル名が違えば別エントリとして追加可（クライアント確認済み）
- 重複チェック用セットはオリジナル＋コピーの両方のチャンネル名を合算する

---

## YouTube URL

- 保存形式は必ず `https://www.youtube.com/channel/UC...` 形式
- `@日本語ハンドル` 形式（例: `@みおんゴルフチャンネル`）は使わない
  - `youtube-api.mjs` の `searchYoutubeChannels` で非ASCII ハンドルは自動フォールバック済み
- 既存行に日本語 URL が混入していたら: `ONLY_NON_ASCII=1 node tools/normalize-youtube-channel-urls.mjs`

## 除外ルール（クライアント指定）

以下は発見しても追加しない:

| 除外対象 | 理由 |
|---|---|
| ボウリング・釣り（フィッシング） | クライアント指定 |
| 海外チャンネル | 対象外 |
| 切り抜きチャンネル | 対象外 |
| 営業禁止と明示しているチャンネル | クライアント指定 |
| 重複チャンネル | 上記ルール参照 |

## スポーツ用品企業の判定

- **チャンネル名にスポーツ用品キーワードが含まれること**（必須）
  - 会社名は検索結果由来のノイズが多いためキーワード判定に使わない
- さらに **法人格あり（株式会社・有限会社等）** または **会社ドメインのメールアドレス** のどちらかを満たすこと
- スポーツ「チーム」「メディア」「個人」はNG、スポーツ「用品」を売る企業がYES

---

## 会社名・代表者名

- 役職（代表取締役、社長等）は除去してから保存
- 括弧内の別表記（英語名、法人番号等）は除去: `株式会社xxx（ISAMI co.）` → `株式会社xxx`
- `(株)` は `株式会社` に変換してから保存
- 後置ノイズを除去: `株式会社xxx　お問い合わせページ` → `株式会社xxx`
- プレフィックスを除去: `社名：株式会社xxx` → `株式会社xxx`
- 「ショップ名・店名・屋号」ラベルも会社名として取得する（法人格がなくても可）
- プラットフォームドメイン（mag2.com、lin.ee、note.com 等）は自社サイトではないため企業情報取得をスキップ

---

## メールアドレス

- 問い合わせフォームURLよりメールアドレスを優先する（クライアント要件）
- 探索優先順位: **特商法ページ > 会社概要 > お問い合わせ > トップページ**
- Gmailなどの個人ドメインは企業メールとして判定しない

---

## チャンネル登録者数・最終投稿日（必須）

- **行を書き込む時点で必ず `getYoutubeChannelMetricsByUrl` を呼んで I列・J列に値を入れる**
- discoverスクリプト・backfillスクリプト問わず、書き込み後に別途補完する運用は禁止
- metrics が取れなかった場合のみ空文字を許容する（その場合は `fill-youtube-channel-metrics.mjs` で後追い補完）

---

## API クォータ管理

- YouTube APIキーは3本体制（primary / fallback1 / fallback2）
- `channels.list`（1単位）を優先し、`search.list`（100単位）は最終手段
- Brave / Google / Firecrawl APIが全て 402/429 になったら直接 HTTP スクレイプに切り替える
- クォータはPT midnight（日本時間09:00）にリセット

## 修正後の記録（必須・自動）

**いかなる修正（コード・データ・書式・スクリプト）を行っても、必ず同じターンで以下を `IMPROVEMENTS.md` に追記すること。ユーザーへの報告より先に記録を完了させる。**

1. **原因究明** — なぜ問題が起きたか（根本原因まで）
2. **再発防止** — 次回同じ問題を防ぐ具体的な対策・ルール変更・コード修正
3. **修正実績** — 何を何に直したか（件数・行番号・値など）

## バッチ修正の手順

1. **必ず `--dry-run` で before/after を確認してから本番実行**
2. 本番実行後に上記「修正後の記録」を即時実施
3. `backfill-company-info.mjs` など補完スクリプトは東たくみ行のみ対象であることを確認してから実行

---

## オリジナルへの転記手順

1. コピーシートで内容を確認する
2. オリジナルシートの既存チャンネル名を取得（重複除外に使う）
3. 未追加行だけを抽出し、コピーシート内の重複も除く
4. オリジナルシートの最終記入行の次から B列（index=1）で追記
   - **必ず事前に周辺行を確認すること**。他の記入者が途中の行をスキップして後ろに書いているケースがある（例: row322が空きなのに row1000から別記入者が書いている）
   - `findStartRow` で求めた行番号が想定外に大きい場合はシート全体をスキャンして空き行を特定する
5. A列には書き込まない
6. 値書き込み後、追記した行の **J・K列（最終投稿日・取得日）に `batchUpdate` でCENTER+MIDDLE書式を明示的に設定する**
   - 理由: オリジナルシートのJ・K書式はセル単位設定。`values.update` では書式が引き継がれず左揃いになる
