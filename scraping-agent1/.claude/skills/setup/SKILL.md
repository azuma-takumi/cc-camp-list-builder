# セットアップスキル

受講生が `scraping-agent` を初めて使うときや、任意 API キーを後から追加するときに、
必要な設定を対話で揃えていくスキル。
詳細手順は `setup/SETUP-FOR-CC.md` にあるので、**このスキル発動時はまずそれを読む**。

---

## 絶対に守るルール

1. **状態チェックを最初に行う**: すでに揃っている設定は絶対に再実行しない
2. **1ステップずつ案内する**: 一度に全部の手順を見せない。ユーザーが「できた」と言うまで次に進まない
3. **ユーザーにファイル編集させない**: クライアントID・シークレット・APIキーは全部チャットに貼り付けてもらい、エージェント側で `.env` に書き込む
4. **途中中断しても、再開できるように動く**: 次に開始されたら、不足分だけ案内する
5. **必須と任意を区別する**: 必須は必ず揃える。任意は「推奨するが、やらなくても先に進める」扱い

---

## ワークフロー

### Step 1. 状態チェック

セットアップがどこまで進んでいるかをまず確認する。以下を順に読む/判定する:

**必須ブロック**:

1. `package.json` が存在するか
2. `node_modules/` が存在するか
3. `.env` が存在し、以下キーが空でないか:
   - `GOOGLE_CLIENT_ID`
   - `GOOGLE_CLIENT_SECRET`
   - `SPREADSHEET_ID`
4. `credentials/tokens.json` が存在するか

**任意(推奨)ブロック**:

5. `.env` に以下が入っているか(別途チェック、欠けても先に進めてよい):
   - `BRAVE_SEARCH_API_KEY`(search-based モード用)
   - `GOOGLE_PLACES_API_KEY`(places モード + Places フォールバック用)

**チェック後、ユーザーに進捗を伝える**:

```text
現在のセットアップ状況:

【必須】
1. 依存パッケージ               <✅済み / ❌未>
2. .env(Google OAuth)           <✅済み / ❌未>
3. スプレッドシート             <✅済み / ❌未>
4. Google 認証(tokens)          <✅済み / ❌未>

【任意(推奨)】
5. Brave Search API             <✅済み / ⏭ 未(後でOK)>
6. Google Places API            <✅済み / ⏭ 未(後でOK)>

<判定サマリ>
```

判定サマリの出し方:

- 必須がどれか ❌ の場合: `必須が <N>件 未完了です。まず必須から進めましょう。`
- 必須は全て ✅ で任意が一つでも未 の場合: `必須は完了しています。任意キーは今追加しますか?後回しでもOKです。`
- 全部 ✅ の場合: `セットアップ完了です。`

### Step 2. 開始時の選択肢(初めて開いた場合)

ユーザーが「セットアップして」「初期設定」「始めたい」で入ってきた場合、
状態チェック結果を表示したあとに **進め方を選ばせる**:

```text
どう進めますか?

1. 全部まとめてセットアップする(必須 + 任意の Brave / Places も一緒に)← 推奨
2. 必須だけ先にセットアップする(任意は後で追加する)
3. 不足分だけ自動で進める(既に揃っているものはスキップ)
```

- `1` → 必須 4 項目 + 任意 2 項目 をすべて実行
- `2` → 必須 4 項目のみ実行し、Step 4-B(任意キー追加の案内)で終わる
- `3` → Step 3 の不足ステップ実行へ

### Step 3. 不足ステップの実行

`setup/SETUP-FOR-CC.md` を Read で開き、そこに書かれている手順に従う。
**スキップできるステップは飛ばし、未完了のステップだけ実行する**。

任意ステップ(2-B / 2-C)は、ユーザーが `1` を選んだ場合のみ実行。
`2` or `3` を選んだ場合は、未完了でもそのまま飛ばし、最後に「後から追加しますか?」と聞く。

手順書との関係:

| セットアップ項目 | 区分 | 担当するステップ (SETUP-FOR-CC.md) |
|---|---|---|
| 依存パッケージ | 必須 | Step 1 |
| Google OAuth クライアント発行 | 必須 | Step 2-A (A-1 〜 A-5) |
| Brave Search API キー取得 | 任意 | Step 2-B |
| Google Places API キー取得 | 任意 | Step 2-C (C-1 〜 C-4) |
| Google 認証 | 必須 | Step 3 |
| スプレッドシート作成 | 必須 | Step 4 |
| 動作確認 | 任意 | Step 5 |
| launchd ラベル(定期実行の識別子)確認 | 任意 | Step 5.5 |

**Step 5.5(launchd ラベル確認)の扱い**:
- 受講生の大半はデフォルト(`local.scraping-agent`)で問題ない
- 必ず 1 回は「変更しますか?」と尋ねる(ユーザーの選択を尊重する方針)
- 変更を希望された場合のみ、ユーザーから受け取った値を `.env` の `LAUNCHD_LABEL_PREFIX` に書き込む

### Step 4. 完了報告

必須がすべて ✅ になったら、任意の充足で分岐する:

#### 4-A. 必須+任意すべて揃っている場合

```text
セットアップ完了です!すべての機能が使えます。

使える機能: places / search-based / list-detail / single の 4 モード、連絡先エンリッチ、定期実行

次にやれること:
1. リサーチを始める(対話しながらスクレイピング)
2. 試しに動作確認する(example.com で簡単なテスト)
3. 詳しい使い方を見る(README.md を表示)

何から始めますか?
```

#### 4-B. 必須のみ揃って、任意が欠けている場合

```text
必須のセットアップが完了しました!

今すぐ使える機能: list-detail / single モードのリサーチ、スケジュール、スクリプト化、一覧/管理

まだ使えない機能:
{未設定の任意キーごとに1行ずつ記載}
  - Brave Search API が未設定 → search-based モード(検索経由のリサーチ)
  - Google Places API が未設定 → places モード(店舗の電話番号取得)、連絡先エンリッチの Places フォールバック

どうしますか?

1. 今すぐ Brave Search API を追加する
2. 今すぐ Google Places API を追加する
3. 両方とも今追加する(推奨)
4. 後で追加する(「開始」と言えばいつでも呼べます)
```

`1` / `2` / `3` → 次セクションの **任意キー追加フロー** に入る。
`4` → ここで完了とし、ユーザーに「リサーチを始めますか?」などを聞く。

## 任意キー追加フロー(後から呼べる、単独起動もOK)

このスキルは以下のケースで呼ばれる:

- Step 4-B で `1/2/3` が選ばれた場合
- `CLAUDE.md` のエントリーポイント判定 B(必須のみ OK)のメニューで「3. 任意キーを追加する」が選ばれた場合
- ユーザーが直接「Brave のキーを追加したい」「Places の API 有効化したい」等と言った場合

### Step N-1. 対象 API を確認

```text
どれを追加しますか?

1. Brave Search API(search-based モード)
2. Google Places API(places モード + 連絡先エンリッチの Places フォールバック)
3. 両方追加する
```

### Step N-2. 対象の手順を実行

- 「Brave」選択 → `setup/SETUP-FOR-CC.md` のステップ 2-B を実行
- 「Places」選択 → `setup/SETUP-FOR-CC.md` のステップ 2-C(C-1 〜 C-4)を実行
- 「両方」選択 → 2-B → 2-C の順で連続実行

途中のキー発行手順は、SETUP-FOR-CC.md の該当セクションをそのまま案内する(このスキルに重複で書かない)。

### Step N-3. 完了報告

```text
{追加した API 名} の設定が完了しました!

これで使える機能:
{追加で使えるようになった機能を記載}

まだ未設定の任意キーがあれば追加しますか?
1. 続けて追加する
2. もう十分です(リサーチを始める)
```

## 特殊ケース

### 「最初からやり直したい」と言われた場合

```text
最初からやり直すには、以下を削除する必要があります:

1. `.env` の該当値をリセット(どれをリセットしますか?)
2. `credentials/tokens.json` を削除(Google 認証をやり直す場合)

以下のどれを削除しますか?

1. Google OAuth 関連だけ(.env のクライアントID/Secret + tokens.json)
2. Brave Search API キーだけ
3. Google Places API キーだけ
4. スプレッドシートID だけ
5. 全部リセット
6. キャンセル
```

選ばれたものだけリセットして、Step 1 の状態チェックから再開する。

### 途中で Claude Code を閉じて、再度起動した場合

ユーザーは何も言わなくてもよい。単に「セットアップして」or「開始」と言えば:

1. Step 1 の状態チェックで進捗が確認できる
2. 完了分はスキップ、未完了分だけ案内される
3. 追加の再実行は発生しない
